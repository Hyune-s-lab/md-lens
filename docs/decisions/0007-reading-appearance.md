# ADR-0007: 읽기 프로필, 텍스트 배율과 가로 폭은 독립적으로 적용한다

- **상태:** 수락됨
- **날짜:** 2026-07-12

## 맥락

MdLens은 GitHub Light와 GitHub Dark의 기본 배경, 본문과 링크 색상을 유지하면서 장문 가독성을 높일 수 있어야 한다. 링크와 제목이 같은 파란색이면 시각적 역할이 겹친다. 새 테마, 읽기 간격, 글꼴, 크기를 하나의 옵션으로 묶으면 각 설정의 의미가 불분명해지고 조합도 불필요하게 늘어난다.

페이지 전체를 확대하면 이미지와 Mermaid 도형까지 함께 커진다. 다이어그램은 제한된 폭에서 다시 배치되거나 글자가 도형을 벗어날 수 있으므로 일반 문서의 글자 크기와 같은 방식으로 확대하면 안 된다.

문서의 최대 가로 폭은 넓은 화면에서 한 줄이 과도하게 길어지는 것을 막아야 하지만, 읽기 프로필의 밀도와 결합하면 Compact와 Spacious의 역할이 불분명해진다. 기술 문서의 코드, 표와 다이어그램에는 일반 장문보다 넓은 공간이 필요하며, 사용자가 화면 전체를 쓰는 선택도 가능해야 한다.

## 결정

- `Compact` 프로필은 GitHub의 기본적인 밀도와 계층을 유지한다.
- `Spacious` 프로필은 행간, 문단 간격과 제목 계층을 넓히고, 테마별 주황색 제목 강조로 파란색 링크와 역할을 구분한다.
- 두 프로필 모두 GitHub의 H1과 H2 하단 구분선을 유지한다.
- 본문 글꼴과 코드 글꼴은 운영체제에 설치된 추천 글꼴 목록에서 독립적으로 선택하며, 기본값이나 사용할 수 없는 글꼴은 시스템 기본 글꼴로 대체한다.
- 글자 크기는 절대 픽셀이 아니라 90%–180% 배율로 저장한다. 배율은 문서 텍스트에만 적용하고 이미지 크기와 Mermaid 도형의 기하는 바꾸지 않는다.
- 문서 최대 폭은 읽기 프로필과 독립적으로 적용한다. 두 프로필 모두 기본적으로 사용 가능한 가로 폭 전체를 쓴다. 필요하면 1152px 권장값에서 시작해 768–1536px 범위의 제한을 64px 단위로 적용할 수 있다. 문서는 좌측 정렬을 유지한다.
- 설정 미리보기는 실제 문서와 같은 번들 렌더러로 고정 Markdown 샘플을 렌더링한다. 설정 창이 열려 있을 때만 JCEF를 유지하고 Mermaid 런타임은 불러오지 않는다.
- 추가 테마와 사용자 CSS는 이후 범위로 남긴다.

## 결과

테마, 읽기 프로필, 타이포그래피와 문서 폭의 역할이 분리된다. 사용자는 익숙한 GitHub 배경, 본문과 링크 색상을 유지하면서 문서 밀도, 글자 크기와 사용 가능한 가로 공간을 독립적으로 조절할 수 있고, Spacious 제목은 링크와 즉시 구분된다. 큰 글자를 선택해도 이미지와 다이어그램 레이아웃은 안정적으로 유지된다. Apply 전 미리보기와 실제 문서가 같은 렌더러를 사용하므로 별도의 스타일 구현이 어긋나지 않는다.
