# ADR-0008: 원격 https 이미지를 허용하고 다이어그램 이미지 실패는 1회 재시도한다

- **상태:** 수락됨
- **날짜:** 2026-07-14

## 맥락

뷰어의 CSP는 `img-src 'self' file: data:`로 원격 이미지를 차단했다. Mermaid 아이콘 노드(`@{ img: "https://..." }`)는 terrastruct, simpleicons 같은 원격 아이콘 CDN을 참조하는 것이 일반적인 사용 패턴인데, 이미지 로드가 차단되면 Mermaid는 해당 다이어그램 전체를 실패 처리한다. 일반 Markdown의 원격 이미지(배지, 스크린샷)도 같은 이유로 표시되지 않았다. 사용자는 이런 다이어그램을 렌더링하기 위해 별도의 Mermaid 플러그인을 유지해야 했다.

원격 이미지를 허용해도 첫 로드에서는 네트워크 타이밍에 따라 이미지 디코드가 간헐적으로 실패할 수 있고(`The source image cannot be decoded.`), 이때도 다이어그램 전체가 실패한다. 같은 문서를 다시 렌더링하면 브라우저 캐시에 적중해 성공한다.

## 결정

- CSP `img-src`에 `https:`를 추가한다. `connect-src`, `script-src` 등 나머지 지시어는 그대로 잠근다. 이미지는 표시 전용이며 스크립트 실행 경로가 아니므로 sanitization 모델은 유지된다.
- 평문 `http:` 이미지는 계속 차단한다.
- Mermaid 다이어그램 렌더링이 이미지 디코드 오류로 실패하면 같은 다이어그램을 1회 재시도한다. 재시도는 브라우저 캐시에 적중하므로 콜드 로드의 간헐 실패를 흡수한다. 파스 오류는 재시도하지 않는다.

## 결과

원격 아이콘을 쓰는 Mermaid 다이어그램과 원격 이미지를 포함한 Markdown 문서가 별도 플러그인 없이 렌더링된다. 렌더러와 Mermaid 런타임은 여전히 번들로 동작하므로 엔진 자체는 오프라인이지만, 문서가 원격 이미지를 참조하면 해당 리소스는 네트워크가 있어야 표시된다. 오프라인에서는 이미지 없는 다이어그램과 문서 본문이 기존과 동일하게 렌더링된다.
